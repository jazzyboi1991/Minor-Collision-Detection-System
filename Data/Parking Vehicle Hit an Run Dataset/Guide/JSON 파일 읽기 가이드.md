# JSON 데이터셋 읽기 가이드

이 문서는 `original_dataset.json` 파일의 구조와 각 데이터 필드의 의미를 설명하며, 영상 분석에 필요한 명도 값 해석 방법을 안내합니다.

---

## 1. 데이터셋 개요
`original_dataset.json`은 영상별 객체(차량) 위치, 사고 정보, 색상 및 명도 통계를 통합한 데이터 파일입니다. 각 영상은 고유한 ID(예: `220510_LA_0000`)를 키로 가집니다.

---

## 2. 필드 상세 설명

### 2.1. `cars` (객체 정보)
영상 내에 존재하는 차량 리스트입니다.
- **`id`**: 차량 고유 식별 번호 (해당 영상 내에서 유효)
- **`bbox`**: 차량의 위치를 나타내는 경계 상자 (Bounding Box)
  - 형식: `[x1, y1, x2, y2]`
  - `(x1, y1)`: 상자의 좌측 상단 좌표
  - `(x2, y2)`: 상자의 우측 하단 좌표

### 2.2. `accidents` (사고 정보)
영상 내에서 발생한 사고(물피도주)에 대한 기록입니다.
- **`vehicle_id`**: 사고를 낸 차량의 `id`
- **`start_frame`**: 사고가 시작되는 시점의 프레임 번호
- **`end_frame`**: 사고 상황이 끝나는 시점의 프레임 번호

### 2.3. 색상 통계 (`r_avg`, `g_avg`, `b_avg`)
전체 영상의 평균 RGB 색상 값입니다. (범위: 0 ~ 255)
- **`r_avg`**: Red 채널 평균
- **`g_avg`**: Green 채널 평균
- **`b_avg`**: Blue 채널 평균

---

## 3. 명도 값(average_brightness) 가이드

`average_brightness`는 영상의 전반적인 밝기를 나타내며, 단순 산술 평균이 아닌 인간의 시각 특성을 반영한 **ITU-R BT.601 Luma 공식**으로 계산되었습니다.

### 3.1. 산출 공식
$$Brightness = (R \times 0.299) + (G \times 0.587) + (B \times 0.114)$$

- **가중치 의미**: 인간의 눈은 초록색(G)에 가장 민감하고 파란색(B)에 가장 둔감하기 때문에, 이를 반영하여 실제 사람이 느끼는 밝기와 유사하게 수치화했습니다.

### 3.2. 환경 판별 기준
명도 수치를 통해 영상의 촬영 환경을 다음과 같이 분류할 수 있습니다.
- **80 이상**: 주간 또는 매우 밝은 실내 조명
- **40 ~ 80**: 일반적인 지하 주차장, 흐린 날 또는 해 질 녘
- **40 이하**: 야간 또는 어두운 저조도 환경 (보정 필요 가능성 높음)

---

## 4. 데이터 활용 팁
1. **사고 시점 추출**: `accidents`의 프레임 정보를 사용하여 원본 영상에서 사고 발생 구간만 잘라내어(Crop) 학습 데이터로 활용할 수 있습니다.
2. **조도 정규화**: `average_brightness`가 낮은 영상(40 이하)은 밝기 보정 필터를 적용하여 객체 인식률을 높이는 전처리 과정을 권장합니다.
3. **환경별 성능 분석**: 명도 값을 기준으로 데이터를 그룹화하여, 야간 환경에서의 모델 탐지 정확도를 별도로 측정할 수 있습니다.
